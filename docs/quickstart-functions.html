Boson Functions quick start guide

This quick start guide demonstrates how you can create, deploy, and modify Boson Functions.

Prerequisites

Before you can complete the steps in the following procedures, you must ensure that:

  • You have access to an OpenShift Container Platform cluster, which has the OpenShift Serverless Operator, Knative Serving and Knative Eventing installed.

  • You have installed the OpenShift oc CLI.

  • You must have installed Docker, and have access to a DockerHub account.

  • You have installed the Knative kn CLI. Installing kn provides the binary that enables use of the kn faas commands. There is no additional setup required to use kn faas commands after kn is successfully installed.

Creating and managing Functions as a Knative service

After kn is installed, you can enter kn faas --help to see available options for using the CLI to create and manage functions.

Creating a function

You can create a function by using the kn faas create command.

Procedure

  1. Create a directory for the function:

    Example commands
    $ mkdir functions
$ cd functions
$ mkdir fn.example.io
$ cd fn.example.io

  2. After you have navigated to the functions directory, create a function:

    $ kn faas create -r <registry> -l <runtime> -t <trigger> -i <image> -n <namespace>
    Example command
    $ kn faas create -r docker.io/test -l node -n default
    Example output
