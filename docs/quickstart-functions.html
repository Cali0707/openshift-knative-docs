Functions Quick Start Guide

You can use this guide to set up the tools required to use Functions on OpenShift Serveless.

Prerequisites

  • Install OpenShift Serverless, including the Knative Serving and Knative Eventing APIs on OpenShift Container Platform.

  • Install the kn CLI.

Installing the CLI

  1. Download the faas CLI:

    $ curl -L -o faas https://github.com/boson-project/faas/releases/download/v0.7.0/faas_linux_amd64 && chmod 755 faas

  2. Move the binary to your $PATH:

    $ sudo mv faas /usr/local/bin

Setting up a container registry

Functions uses OCI container images for deployment, typically referred to as Docker container images. To enable the faas CLI to manage these container images, you must have access to a container registry and a repository in that registry.

  1. Login to Docker hub:

    $ docker login -u <username> -p <password>

  2. Many faas CLI commands require a --repository argument.

    You can set the FAAS_REPOSITORY environment variable so that you can omit this parameter when using the CLI:

    $ export FAAS_REPOSITORY=docker.io/<username>/<repository_name>

Create a Functions project

  1. Create a project directory:

    $ mkdir fn.example.io

  2. Navigate to the new directory:

    $ cd fn.example.io

  3. Create the project files, build a container, and deploy the function as a Knative service:

    $ faas create -l node -y
